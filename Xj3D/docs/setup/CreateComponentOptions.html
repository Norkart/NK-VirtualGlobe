The EAI BrowserFactory.createComponent and the SAI BrowserFactory.createX3DComponent methods both allow user code to specify configuration options. While the two methods differ in how the options are passed in (the EAI requires the options to be encoded as Option=Value strings and the SAI requires the options to be stored in a Map object) the same configuration options are used. These options are inherently non-standard since the VRML and X3D specifications provide no list of standard configuration parameters.
Xj3D_NavbarShown
true or false to control the display the toolbar with navigational controls and viewpoint selection
Xj3D_NavbarPosition
top or bottom to specify whether the navigation toolbar is on the top or bottom of the browser panel
Xj3D_LocationShown
true or false to control the display of the toolbar with the world URL and load URL button
Xj3D_ViewpointShown
true or false to control the display of the viewpoint controls in the navigation toolbar
Xj3D_LocationPosition
top or bottom to control whether the location toolbar is on the top or bottom of the browser panel
Xj3D_LocationReadOnly
true or false to control whether the address field of the location toolbar is editable and by extension whether the load URL button on the lcoation toolbar is enabled
Xj3D_ShowConsole
true or false to control whether the browser console window (used to output debugging information) is displayed when the panel is created
Xj3D_ServerPort
Integer specifying the port to listen on in support of the getBrowser(InetAddress,port) method. If not specified the browser does not listen on any port. In the M10 release only EAI is supported.
Xj3D_BrowserSkin
For the SAI createX3DComponent method only and allows specifying a Map containing browser skin parameters. See the browser skin documentation for details.
